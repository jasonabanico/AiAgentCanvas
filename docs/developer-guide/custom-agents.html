Adding Custom Agents

A custom agent in AI Agent Canvas is a project under src/Custom/ that pairs domain-specific tools with a persona that tells the LLM how to use them. This is the primary extension point -- you add tools for your domain and seed a persona that gives the agent its role and personality.

How It Works

The Core framework collects all registered IReadOnlyList<AITool> services and passes them to the ChatClientAgent. Custom agents register their tools and optionally seed a persona via IPersonaSeed. At startup, seeded personas are saved to agent-data/personas/ if they don't already exist. Users can activate them at runtime by saying "switch to the [persona-name] persona".

Step-by-Step Guide

Step 1: Create a Project Folder

Create a new folder under src/Custom/:

src/Custom/MyAgent/

Step 2: Create the .csproj

Add package references for Microsoft.Extensions.AI (for tool creation) and a project reference to Abstractions (for persona seeding):

<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <TargetFramework>net9.0</TargetFramework>
    <ImplicitUsings>enable</ImplicitUsings>
    <Nullable>enable</Nullable>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Microsoft.Extensions.AI" Version="10.7.0" />
    <PackageReference Include="Microsoft.Extensions.DependencyInjection.Abstractions" Version="10.0.9" />
  </ItemGroup>

  <ItemGroup>
    <ProjectReference Include="..\..\AiAgentCanvas.Abstractions\AiAgentCanvas.Abstractions.csproj" />
  </ItemGroup>

</Project>

Step 3: Create a Tool Provider

Create a class that returns the tools the LLM can call. Use [Description] attributes so the model understands each parameter:

using System.ComponentModel;
using System.Text.Json;
using Microsoft.Extensions.AI;

namespace MyAgent;

public sealed class MyAgentToolProvider
{
    public IReadOnlyList<AITool> GetTools() =>
    [
        AIFunctionFactory.Create(SearchKnowledgeBase, "search_kb",
            "Search the product knowledge base"),
        AIFunctionFactory.Create(LookupOrder, "lookup_order",
            "Look up an order by order number"),
    ];

    [Description("Search the product knowledge base")]
    private static string SearchKnowledgeBase(
        [Description("Search query")] string query)
    {
        // Your implementation here
        return JsonSerializer.Serialize(new { results = new[] { "..." } });
    }

    [Description("Look up an order by order number")]
    private static string LookupOrder(
        [Description("The order number to look up")] string orderNumber)
    {
        return JsonSerializer.Serialize(new { order = orderNumber, status = "shipped" });
    }
}

Step 4: Create a Service Extension

Register your tools and seed a persona that tells the LLM its role and which tools to use:

using AiAgentCanvas.Abstractions;
using Microsoft.Extensions.AI;
using Microsoft.Extensions.DependencyInjection;

namespace MyAgent;

public static class MyAgentServiceExtensions
{
    public static IServiceCollection AddMyAgent(this IServiceCollection services)
    {
        // Register tools
        services.AddSingleton<MyAgentToolProvider>();
        services.AddSingleton<IReadOnlyList<AITool>>(sp =>
            sp.GetRequiredService<MyAgentToolProvider>().GetTools());

        // Seed a persona
        services.AddSingleton<IPersonaSeed>(new PersonaSeed(
            name: "customer-support",
            description: "Customer support assistant for Contoso Electronics",
            instructions: """
                You are a customer support assistant for Contoso Electronics.

                Use these tools:
                - search_kb: Search the product knowledge base
                - lookup_order: Look up an order by order number

                Guidelines:
                - Always greet the customer warmly
                - Look up order details before making claims about order status
                - Never share internal pricing or margin information
                """));

        return services;
    }
}

Step 5: Add ProjectReference in AiAgentCanvas.Web.csproj

Open src/AiAgentCanvas.Web/AiAgentCanvas.Web.csproj and add a reference to your project:

<ProjectReference Include="..\Custom\MyAgent\MyAgent.csproj" />

Step 6: Add to AiAgentCanvas.sln

Add the project to the solution under the Custom solution folder:

dotnet sln AiAgentCanvas.sln add src/Custom/MyAgent/MyAgent.csproj --solution-folder Custom

Step 7: Wire Up in Program.cs

using MyAgent;

builder.Services.AddMyAgent();

Tools are immediately available to the agent. The persona is saved to agent-data/personas/customer-support.md on first startup. Users activate it with: "switch to the customer-support persona".

Full Reference: HelloWorldAgent

The HelloWorldAgent in src/Custom/HelloWorldAgent/ is the built-in sample. It demonstrates the complete pattern with three files:

HelloWorldToolProvider.cs

Registers three tools: hello_greet, hello_roll_dice, and hello_coin_flip.

public sealed class HelloWorldToolProvider
{

        AIFunctionFactory.Create(RollDice, "hello_roll_dice",

            "Flip a coin and return heads or tails"),
    ];

    private static string Greet(string name) => /* ... */;
    private static string RollDice(int count = 1, int sides = 6) => /* ... */;
    private static string CoinFlip() => /* ... */;
}

HelloWorldServiceExtensions.cs

Registers the tools and seeds a persona that gives the agent its personality:

public static IServiceCollection AddHelloWorldAgent(this IServiceCollection services)
{
    // Register tools
    services.AddSingleton<HelloWorldToolProvider>();
    services.AddSingleton<IReadOnlyList<AITool>>(sp =>
        sp.GetRequiredService<HelloWorldToolProvider>().GetTools());

    // Seed a persona
    services.AddSingleton<IPersonaSeed>(new PersonaSeed(
        name: "hello-world",
        description: "A friendly demo agent that greets users, rolls dice, and flips coins",
        instructions: """
            You are a friendly, enthusiastic demo assistant for AI Agent Canvas.

            You have these tools -- use them whenever relevant:
            - hello_greet: Greet users by name with a warm welcome
            - hello_roll_dice: Roll dice for games, decisions, or fun
            - hello_coin_flip: Flip a coin for quick yes/no decisions

            Keep responses short, upbeat, and show off what tools can do.
            When someone first talks to you, greet them and offer to roll dice or flip a coin.
            """));

    return services;
}

Wired in Program.cs with a single line: builder.Services.AddHelloWorldAgent();

Persona Seeding

The IPersonaSeed interface lets custom agents ship their own personas. At startup, AddAiAgentCanvasPersonas() resolves all registered IPersonaSeed services and saves any that don't already exist on disk. This means:

  • Personas are created automatically on first run
  • Manual edits to the persona file are preserved (seeds don't overwrite)
  • Users activate seeded personas the same way as manually created ones
// Register a persona seed in your service extension
services.AddSingleton<IPersonaSeed>(new PersonaSeed(
    name: "my-persona",
    description: "Short description for listing",
    instructions: "Full system prompt instructions..."));

Switching Between Agents

Users switch agents at runtime via personas -- no restart needed. Say "switch to the hello-world persona" or "switch to the customer-support persona". Say "switch to default" to return to the base system prompt. See Agent Data for details.

Testing Your Agent

  1. Build the solution: dotnet build AiAgentCanvas.sln
  2. Run the backend: cd src/AiAgentCanvas.Web && dotnet run
  3. Run the frontend: cd frontend && npm run dev
  4. Open the CopilotKit UI and interact with your agent
  5. Check the console logs for tool calls and governance events
