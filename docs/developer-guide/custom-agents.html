Adding Custom Agents

Custom agents in AI Agent Canvas are lightweight projects that provide specialized system prompts. An agent does not need to implement any interface or reference any framework project -- it can be a pure class library containing only a prompts class. Tools are provided automatically by the DynamicToolRegistry and startup-registered tool providers.

How Agents Get Tools

A common question is: how does a custom agent access tools? The answer is that it does not manage tools at all. The Core framework collects all registered IReadOnlyList<AITool> services and passes them to the ChatClientAgent. Your agent's system prompt simply instructs the LLM which tools to use and how to use them. The tools themselves are registered separately (see Adding Custom MCP Connections).

Step-by-Step Guide

Step 1: Create a Project Folder

Create a new folder under src/Custom/:

src/Custom/MyAgent/

Step 2: Create the .csproj

Create a minimal project file. For a pure prompts agent, no package references are needed:

<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <TargetFramework>net9.0</TargetFramework>
    <ImplicitUsings>enable</ImplicitUsings>
    <Nullable>enable</Nullable>
  </PropertyGroup>

</Project>

Step 3: Create the Agent Prompts Class

Create a static class with a GetSystemPrompt() method that returns the system prompt for your agent:

namespace MyAgent;

public static class MyAgentPrompts
{
    public static string GetSystemPrompt() => """
        You are a customer support assistant for Contoso Electronics.

        You have access to the following tools:
        - search_knowledge_base: Search the product knowledge base
        - lookup_order: Look up an order by order number
        - create_ticket: Create a support ticket

        Guidelines:
        - Always greet the customer warmly
        - Look up order details before making claims about order status
        - If you cannot resolve an issue, create a support ticket
        - Never share internal pricing or margin information
        - Escalate billing disputes to a human agent
        """;
}

The system prompt should:

  • Describe the agent's role and personality
  • List the tools it should use (these are registered separately)
  • Provide behavioral guidelines and constraints
  • Specify what the agent should and should not do

Step 4: Add ProjectReference in AiAgentCanvas.Web.csproj

Open src/AiAgentCanvas.Web/AiAgentCanvas.Web.csproj and add a reference to your project:

<ItemGroup>
  <ProjectReference Include="..\Custom\MyAgent\MyAgent.csproj" />
</ItemGroup>

Step 5: Add to AiAgentCanvas.sln

Add the project to the solution under the Custom solution folder:

dotnet sln AiAgentCanvas.sln add src/Custom/MyAgent/MyAgent.csproj --solution-folder Custom

Step 6: Wire Up in Program.cs

Use your agent's system prompt when configuring AddAiAgentCanvas():

using MyAgent;

builder.Services.AddAiAgentCanvas(builder.Configuration, options =>
{
    options.AgentName = "CustomerSupport";
    options.AgentDescription = "A customer support assistant for Contoso Electronics.";
    options.SystemPrompt = MyAgentPrompts.GetSystemPrompt();
});

Full Reference: HelloWorldAgent

The HelloWorldAgent in src/Custom/HelloWorldAgent/ is the built-in sample. It has exactly two files.

HelloWorldAgent.csproj

<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <TargetFramework>net9.0</TargetFramework>
    <ImplicitUsings>enable</ImplicitUsings>
    <Nullable>enable</Nullable>
  </PropertyGroup>

</Project>

No package references. No project references. Just a net9.0 class library.

HelloWorldAgentPrompts.cs

namespace HelloWorldAgent;

public static class HelloWorldAgentPrompts
{
    public static string GetSystemPrompt() => """
        You are a helpful stock market assistant.

        You have access to the following tools:
        - stock_quote: Get the current price for a ticker from Yahoo Finance
        - stock_history: Get historical stock data from Yahoo Finance with configurable range
        - edgar_company_facts: Fetch SEC EDGAR financial data for a company

        When a user asks about a stock, use the appropriate tools to fetch live data,
        then summarize the results in a clear, concise response. Use specific numbers.
        If the user doesn't specify a ticker, ask them which stock they're interested in.
        """;
}

The tools referenced in the prompt (stock_quote, stock_history, edgar_company_facts) are provided by the MCP.MarketData custom project. They are registered separately via builder.Services.AddMarketDataTools() in Program.cs.

Switching Between Agents

To switch between agents, change the SystemPrompt in the AddAiAgentCanvas() call in Program.cs. You can also use the Personas agent data domain to switch prompts at runtime without restarting the server -- see Agent Data.

Agents with Custom Tools

If your agent needs custom tools (data connections, API integrations), create them as a separate Custom project following the Adding Custom MCP Connections guide. The separation is intentional: agent prompts define behavior, while tool providers define capabilities.

However, if you prefer to keep tools and prompts in the same project, you can. Just add the Microsoft.Extensions.AI package reference and follow the tool provider pattern:

// MyAgent/MyAgentToolProvider.cs
using Microsoft.Extensions.AI;

namespace MyAgent;

public sealed class MyAgentToolProvider
{
    public IReadOnlyList<AITool> GetTools()
    {
        return
        [
            AIFunctionFactory.Create(SearchKnowledgeBase, "search_knowledge_base",
                "Search the product knowledge base"),
        ];
    }

    private string SearchKnowledgeBase(string query)
    {
        // Your implementation here
        return "...";
    }
}
// MyAgent/MyAgentServiceExtensions.cs
using Microsoft.Extensions.AI;
using Microsoft.Extensions.DependencyInjection;

namespace MyAgent;

public static class MyAgentServiceExtensions
{
    public static IServiceCollection AddMyAgent(this IServiceCollection services)
    {
        services.AddSingleton<MyAgentToolProvider>();
        services.AddSingleton<IReadOnlyList<AITool>>(sp =>
            sp.GetRequiredService<MyAgentToolProvider>().GetTools());
        return services;
    }
}

Then in Program.cs:

builder.Services.AddMyAgent();

Testing Your Agent

  1. Build the solution: dotnet build AiAgentCanvas.sln
  2. Run the backend: cd src/AiAgentCanvas.Web && dotnet run
  3. Run the frontend: cd frontend && npm run dev
  4. Open the CopilotKit UI and interact with your agent
  5. Check the console logs for tool calls and governance events
