AI/LLM Security Libraries for .NET 9

Research for AgentOpsHub -- OWASP Top 10 for LLM Applications mitigation. June 2026.

1. Prompt Injection Detection / Prevention

OWASP LLM01 -- The #1 risk. Libraries that detect direct and indirect prompt injection in user input or retrieved context.

AgentGuard

.NET Native Open Source MIT
NuGet: AgentGuard, AgentGuard.Onnx | GitHub | Docs

22 built-in guardrail rules. Prompt injection detection at 6 tiers: regex patterns, bundled ONNX model (StackOne Defender MiniLM-L6-v2, ~22MB, ~8ms inference, F1 ~0.97), DeBERTa, remote ML classifier, Azure Prompt Shields integration, and LLM-as-judge. Also covers PII, secrets, content safety, topic enforcement, tool-call guardrails (SQL/code injection, SSRF), RAG chunk filtering, and indirect prompt injection in tool results.

Integrates with Microsoft.Extensions.AI IChatClient as a decorator, or with Microsoft Agent Framework via .WithGuardrails(). Split into 8 NuGet packages. Targets .NET 10 (preview), likely compatible with .NET 9.

Best fit for AgentOpsHub -- .NET-native, covers most OWASP LLM categories in one library.

Microsoft Agent Governance Toolkit

.NET Native Open Source MIT
NuGet: Microsoft.AgentGovernance, Microsoft.AgentGovernance.Extensions.ModelContextProtocol | GitHub

Microsoft's official open-source runtime security toolkit for AI agents. Claims coverage of all 10 OWASP Agentic AI risks. Stateless policy engine (Agent OS) with sub-millisecond enforcement. Supports YAML rules, OPA Rego, and Cedar policy languages. Includes Agent Mesh (cryptographic identity, trust scoring), Agent Runtime (execution rings, saga orchestration), and Agent SRE (circuit breakers, chaos engineering).

Available in .NET, Python, TypeScript, Rust, Go. Integrates with Semantic Kernel and MCP servers. Released April 2026.

Strong fit -- enterprise-grade, Microsoft-backed, .NET-native with MCP integration.

LLM Guard (ProtectAI)

Python Only Open Source
PyPI: llm-guard | GitHub | Docs

Comprehensive input/output scanners: prompt injection (DeBERTa-based), jailbreak, invisible text, PII, toxicity, ban topics, regex, secrets, language detection. Self-hosted, data never leaves infrastructure. Python 3.10+ only.

No .NET SDK. Would need to be deployed as a sidecar/microservice and called via HTTP from .NET.

Good capabilities but Python-only. Consider as a sidecar if AgentGuard doesn't meet needs.

NVIDIA NeMo Guardrails

Python Only Open Source
PyPI: nemoguardrails | GitHub

Programmable guardrails with Colang language for defining conversation flows. Topical, safety, and security rails. Integrates with LangChain, LlamaIndex. Async-first Python library, no .NET SDK. LangChain dependency now optional.

Python-only, no .NET path. Overkill for .NET apps -- AgentGuard covers the same ground natively.

Lakera Guard (now Cisco)

REST API Paid Service
API: POST https://api.lakera.ai/v2/guard | Docs

Commercial prompt injection/jailbreak detection API. 98%+ detection, sub-50ms latency, 100+ languages. Acquired by Cisco (May 2025). No .NET SDK -- REST API only, OpenAI-compatible message format. Requires paid subscription.

Not open source, paid API. REST-callable from .NET but adds vendor dependency + cost.

guardrails-ai

Python Only Open Source
PyPI: guardrails-ai | GitHub | Site

Python framework for adding structural and semantic validation to LLM outputs. Validator hub with community validators. No .NET SDK whatsoever.

Python-only. The .NET equivalent is AgentGuard.

2. PII / Secret Redaction

OWASP LLM06 (Sensitive Information Disclosure) -- Detect and redact PII, API keys, secrets from prompts and responses.

Microsoft Presidio

Python Only REST API Available Open Source
PyPI: presidio-analyzer, presidio-anonymizer | GitHub | Docs

Microsoft's flagship PII detection/anonymization framework. NER + regex + contextual analysis. Supports text, images, structured data. 50+ entity types. Can be deployed as a dockerized REST API, making it callable from .NET. No native .NET NuGet package -- Python only.

Actively maintained (2024-2025 updates). Enterprise-proven. Can run as a sidecar container alongside .NET services.

Gold standard for PII, but Python-only. Deploy as Docker sidecar + HTTP client from .NET. AgentGuard bundles ~50 PII entity types natively in .NET.

AgentGuard.Pii

.NET Native Open Source
NuGet: AgentGuard.Pii | GitHub

~50 PII entity types with reversible encryption, de-identification, and structured data support. Runs inline in .NET process -- no sidecar needed. Part of the AgentGuard ecosystem.

Best .NET-native PII option. Use this instead of Presidio unless you need Presidio's NER depth.

3. Input/Output Guardrails Frameworks

OWASP LLM01, LLM02, LLM05 -- Frameworks that wrap LLM calls with safety checks on both input and output.

Semantic Kernel Filters

.NET Native Built-in
NuGet: Microsoft.SemanticKernel | Docs

Semantic Kernel has built-in prompt injection mitigations: auto HTML-encoding of inserted content (treats all content as untrusted by default), filter pipeline for pre/post LLM call hooks. Filters can integrate with Azure Prompt Shields or custom detection logic. Note: CVE-2026-25592 exposed RCE via prompt injection in SessionsPythonPlugin -- patched in v1.71.0+.

Filters are extensible: IPromptRenderFilter, IFunctionInvocationFilter, IAutoFunctionInvocationFilter. You can inject security checks at each stage.

Already in your stack if using Semantic Kernel. Combine with AgentGuard for deeper detection.

Microsoft.Extensions.AI Middleware Pipeline

.NET Native Built-in
NuGet: Microsoft.Extensions.AI | Docs

The IChatClient decorator pattern allows wrapping any chat client with middleware. No built-in security middleware ships in the box, but AgentGuard plugs directly into this pipeline as an IChatClient decorator. This is the standard .NET extensibility point for adding security checks.

No built-in security filters, but provides the pipeline architecture. AgentGuard is the security layer for it.

4. Rate Limiting

OWASP LLM04 (Denial of Service) -- Throttle request volume to LLM endpoints.

Microsoft.AspNetCore.RateLimiting (Built-in)

.NET Native Built-in
Part of ASP.NET Core (.NET 7+) | Docs

Built into ASP.NET Core since .NET 7. Four algorithms: fixed window, sliding window, token bucket, concurrency limiter. Policy-based, applied per-endpoint or globally. No extra NuGet needed for .NET 9.

Use this. Built-in, no dependencies, fully supported. Apply to your /api/copilotkit endpoint.

AspNetCoreRateLimit

.NET Native Open Source
NuGet: AspNetCoreRateLimit | GitHub

Third-party rate limiting with IP-based and client-based policies. Supports Redis/SQL Server distributed stores. More flexible configuration than built-in. However, the built-in middleware has caught up significantly in .NET 7+.

Still useful for Redis-backed distributed limiting. Otherwise prefer the built-in middleware.

5. Content Safety

OWASP LLM02 (Insecure Output Handling), LLM05 -- Detect toxic, harmful, or inappropriate content in LLM I/O.

Azure AI Content Safety

.NET Native SDK Paid Azure Service
NuGet: Azure.AI.ContentSafety (v1.0.0) | NuGet | Docs

Official .NET SDK for Azure AI Content Safety. Detects sexual content, violence, hate, self-harm with multi-severity levels across text and images. Prompt Shields feature detects prompt injection attacks. Requires Azure subscription. API versions: 2024-09-01 and later supported.

Production-grade, .NET SDK available. Costs money but integrates perfectly with Azure-hosted apps.

6. Audit Logging

OWASP LLM09 (Overreliance), LLM10 -- Security-aware structured logging with PII masking.

Serilog.Enrichers.Sensitive

.NET Native Open Source
NuGet: Serilog.Enrichers.Sensitive (v2.1.0, Aug 2025) | GitHub | NuGet

Serilog enricher that masks sensitive data in log messages and properties. Built-in masking for email addresses, IBANs, credit card numbers. Extensible with custom masking operators. Supports sensitive-area scoping. 7.6K downloads/day. Actively maintained.

Good fit for ensuring PII doesn't leak into logs. Pair with structured audit logging.

7. MCP SDK Security

Security features in the Model Context Protocol C# SDK.

ModelContextProtocol C# SDK

.NET Native Open Source
NuGet: ModelContextProtocol | GitHub

The 2025-06-18 spec introduced OAuth 2.1 authentication, Protected Resource Metadata, WWW-Authenticate challenges, authorization filters, and per-session security config. The 2025-11-25 spec added step-up authorization and enterprise managed authorization (RFC 8693 token exchange). Defense against session ID abuse (leaked session IDs can't DoS original owner).

Integrates with ASP.NET Core authentication middleware (JWT Bearer, OAuth, OIDC). No content-level security (no prompt injection detection) -- that's left to the application layer.

Good transport-level security (auth, sessions). No content security -- pair with AgentGuard or Content Safety.

Summary: Recommended Stack for AgentOpsHub

Recommended .NET-native security stack (all open source, no paid APIs required):
  1. AgentGuard -- Primary guardrails layer (prompt injection, PII, secrets, content safety, tool-call validation). Plugs into IChatClient pipeline.
  2. Microsoft.AgentGovernance -- Policy enforcement, identity, execution sandboxing. Covers OWASP Agentic Top 10. Has MCP integration package.
  3. Microsoft.AspNetCore.RateLimiting -- Built-in rate limiting on the /api/copilotkit endpoint.
  4. Serilog.Enrichers.Sensitive -- PII masking in audit logs.
  5. Semantic Kernel Filters -- If using SK, leverage IPromptRenderFilter for additional pre-LLM checks.
  6. Azure AI Content Safety -- Add if on Azure and budget allows, for production-grade content moderation.
